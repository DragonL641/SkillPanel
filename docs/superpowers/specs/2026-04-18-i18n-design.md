# SkillPanel UI 国际化 (i18n) 设计文档

## 需求摘要

为 SkillPanel 前端 UI 添加国际化支持，覆盖中文（zh-CN）和英文（en）两种语言。仅翻译前端 UI 文本，后端 API 错误消息和 LLM 分析 prompt 保持不变。

## 技术选型

**react-i18next** — React 生态最成熟的 i18n 方案。

依赖：
- `i18next` — i18n 核心
- `react-i18next` — React 绑定（`useTranslation` hook）
- `i18next-browser-languagedetector` — 浏览器语言检测 + localStorage 持久化

## 架构设计

### 初始化 (`src/i18n.ts`)

```ts
import i18n from 'i18next'
import { initReactI18next } from 'react-i18next'
import LanguageDetector from 'i18next-browser-languagedetector'

import zhCN from '../../public/locales/zh-CN/translation.json'
import en from '../../public/locales/en/translation.json'

i18n
  .use(LanguageDetector)
  .use(initReactI18next)
  .init({
    resources: { 'zh-CN': { translation: zhCN }, en: { translation: en } },
    fallbackLng: 'zh-CN',
    interpolation: { escapeValue: false },
    detection: {
      order: ['localStorage', 'navigator'],
      lookupLocalStorage: 'skillpanel-lang',
      caches: ['localStorage'],
    },
  })

export default i18n
```

### 入口加载 (`src/main.tsx`)

在 `ReactDOM.createRoot()` 之前 import `./i18n`，确保 i18n 在 React 渲染前初始化。

### 语言检测优先级

1. `localStorage` 中的 `skillpanel-lang`（用户手动覆盖）
2. `navigator.language`（浏览器设置）
3. 回退到 `zh-CN`

### 语言切换组件

在 App header 右上角（设置按钮旁）添加 `LangSwitch` 组件：

- 显示当前语言：`中` 或 `EN`
- 点击切换 `zh-CN` ↔ `en`
- 调用 `i18n.changeLanguage()` 自动持久化到 localStorage

### index.html 处理

`index.html` 的 `<html lang="zh-CN">` 改为 `<html lang="">`，i18n 初始化后通过 `document.documentElement.lang = i18n.language` 动态设置。

## 组件改造策略

### 改造范围

13 个前端文件，~95 个字符串。

### 改造规则

1. 所有硬编码中文替换为 `t('key')` 调用
2. 带变量的字符串使用 i18next 插值：`t('skill.confirmDelete', { name })`
3. `aria-label` 属性也需要翻译
4. 日期格式化改为 `new Date().toLocaleDateString(i18n.language)`
5. hooks 中可使用 `useTranslation()` 获取 `t` 函数

### 涉及文件

| 文件 | 字符串数 | 改造要点 |
|------|---------|---------|
| `App.tsx` | ~8 | 搜索框、加载态、空状态、引入 LangSwitch |
| `StatsRow.tsx` | ~8 | 统计标签和描述 |
| `ConfigModal.tsx` | ~20 | 表单标签、提示、按钮（最复杂） |
| `SkillCard.tsx` | ~10 | 状态标签、确认对话框 |
| `AnalysisPanel.tsx` | ~8 | 分析状态和时间戳、日期 locale |
| `PluginPanel.tsx` | ~10 | 更新状态、空状态、日期 locale |
| `DirTree.tsx` | ~4 | 批量操作、空状态 |
| `DirPicker.tsx` | ~8 | 浏览器导航 |
| `ProjectSidebar.tsx` | ~8 | 项目列表操作 |
| `ProjectSkillView.tsx` | ~10 | 项目技能管理 |
| `AddSkillModal.tsx` | ~8 | 添加流程 |
| `TabSwitch.tsx` | 3 | tab 标签 |
| `useSkills.ts` | ~6 | 错误消息 fallback |
| `usePlugins.ts` | 1 | 错误消息 fallback |

### 不改的部分

- 后端 API 错误消息（保持英文）
- `api/client.ts` 中的开发错误消息（保持英文）
- LLM 分析 prompt（保持中文）
- 后端测试不受影响

## 测试策略

- 现有后端测试不受影响
- 新增一个测试：验证 `zh-CN/translation.json` 和 `en/translation.json` 的 key 集合完全一致，防止翻译遗漏
- 不为 react-i18next 库本身写测试
